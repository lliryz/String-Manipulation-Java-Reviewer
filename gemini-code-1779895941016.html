1. 🔹 String Fundamentals

Strings in Java are objects that represent sequences of characters.

Declaration

String s1 = "hello";
String s2 = new String("hello");
Key Property: Strings are immutable.
Any modification creates a new object in memory rather than altering the existing one.
💡 Practice Problem

What happens to the original string s1 after executing: s1.toUpperCase();?

Nothing. Because strings are immutable, the original s1 remains unchanged. To capture the uppercase version, you must assign it to a variable: s1 = s1.toUpperCase();

2. 🔹 Basic Operations

Length and Characters

String s = "hello";

s.length();      // 5
s.charAt(1);     // 'e' (Zero-based indexing)

Substring

Remember that the end index is exclusive.

String s = "programming";

s.substring(0, 4);   // "prog"
s.substring(4);      // "ramming"
💡 Practice Problem

Write code to extract the word "gram" from the string "programming".

String s = "programming";
String result = s.substring(3, 7);
System.out.println(result); // Prints "gram"

3. 🔹 Searching in Strings

Method Purpose
indexOf() Returns the index of the first occurrence
lastIndexOf() Returns the index of the last occurrence
contains() Checks existence (returns boolean) 
String s = "banana";

s.indexOf("a");      // 1
s.contains("nan");   // true
💡 Practice Problem

How do you find the index of the second occurrence of the letter "a" in "banana"?

Use the overloaded version of indexOf that takes a starting position.

String s = "banana";
int firstA = s.indexOf("a"); // 1
int secondA = s.indexOf("a", firstA + 1); // 3

4. 🔹 String Comparison

Method Function
equals() Exact match (case-sensitive)
equalsIgnoreCase() Ignore case match
compareTo() Lexicographic comparison (returns int) 
"abc".equals("abc");   // true
"abc".compareTo("abd"); // negative integer
Important Rule:
== compares memory references. equals() compares the actual string content. Always use equals() for content comparison.
💡 Practice Problem

What is the output of the following code and why?

String a = new String("test");
String b = new String("test");
System.out.println(a == b);
System.out.println(a.equals(b));

Output:
false
true

Why: a == b is false because they are different objects in memory (created via the new keyword). a.equals(b) is true because their character sequences are identical.

6. 🔹 Tokenization

Tokenization = splitting a string into parts (tokens).

A. Using split() (Primary Method)

Key point: split() uses Regular Expressions (Regex).
String s = "a,b,c";
String[] arr = s.split(",");

// Output array contains: ["a", "b", "c"]

B. Using StringTokenizer (Legacy)

import java.util.StringTokenizer;

String s = "a b c";
StringTokenizer st = new StringTokenizer(s);

while (st.hasMoreTokens()) {
    System.out.println(st.nextToken());
}

// With explicit delimiter
StringTokenizer st2 = new StringTokenizer("a,b,c", ",");

Comparison

Feature split() StringTokenizer
Uses regex Yes No
Modern Yes No
Flexibility High Low

7. 🔹 Delimiters

A delimiter separates values in a string.

Type Example
comma a,b,c
space a b c
semicolon a;b;c
pipe a|b|c
multiple spaces "a b"

Examples

// Single delimiter
String s1 = "a,b,c";
s1.split(",");

// Multiple delimiters using Regex Character Class []
String s2 = "a,b;c|d";
String[] arr2 = s2.split("[,;|]");

// Multiple spaces
String s3 = "Java   Python    C++";
String[] arr3 = s3.split("\\s+");

8. 🔹 Regex (Regular Expressions)

Regex is a pattern language for matching strings.

Core Symbols

SymbolMeaning
.any character
*0 or more
+1 or more
?optional
[]character set
[^]negation (not in set)
()grouping

Predefined Classes

RegexMeaning
\\ddigit
\\Dnon-digit
\\swhitespace
\\Snon-whitespace
\\wword character
\\Wnon-word
Java Escape Rule: Backslashes must be doubled in Java Strings.
Raw Regex: \d ➡ Java String: "\\d"

Usage Examples

// Validation
"12345".matches("\\d+");  // true

// Split by digits
"a1b2c".split("\\d");     // ["a", "b", "c"]

9. 🔹 Replace Operations

Java provides three primary replacement methods:

A. replace() (Literal)

String s = "java java";
s.replace("java", "code"); // "code code"

B. replaceAll() (Regex-based)

String s = "a1b2c3";
s.replaceAll("\\d", "#");  // "a#b#c#"

C. replaceFirst() (Regex-based)

String s = "a1b1b1";
s.replaceFirst("1", "X");  // "aXb1b1"

Comparison

MethodUses RegexBehavior
replace()Noliteral replace all occurrences
replaceAll()Yesreplace all regex matches
replaceFirst()Yesreplace only first regex match

10. 🔹 Combined Patterns

Normalize and Tokenize

A common task is cleaning up messy delimiters before splitting.

String s = "a,,b;;c|d";

// 1. Replace 1 or more (+) messy delimiters with a standard comma
s = s.replaceAll("[,;|]+", ","); 

// 2. Split safely
String[] arr = s.split(",");

Remove Non-Digits (Data Cleaning)

String s = "a1b2c3";
s = s.replaceAll("\\D", ""); // Remove all non-digits

// Result: "123"
💡 Practice Problem

You have a phone number formatted as "(555) 123-4567". Write a single line of code using replaceAll() to strip all non-numeric characters so the result is just "5551234567".

String phone = "(555) 123-4567";
phone = phone.replaceAll("\\D", "");
System.out.println(phone); // Prints "5551234567"

11. 🔹 Key Takeaways

  • Strings are immutable. Modification methods return entirely new Strings.
  • Use equals() for comparison, not ==.
  • split() is regex-based. Remember this when splitting by special regex characters.
  • \\s+ handles multiple unknown spaces efficiently.
  • StringTokenizer is legacy. Prefer split() in modern Java.
  • replace() is literal, replaceAll() is regex-based.
  • Regex is essential for parsing and validation tasks.
